ESP32 Smart Irrigation Monitoring & Control System

A five-node IoT network that monitors soil and environmental conditions in real time and gives growers role-based control over irrigation, instead of watering on a fixed schedule.

ESP32 Flask MySQL Arduino C/C++ Soil Moisture Sensor Ultrasonic Sensor DHT22 Sensor PIR Motion Sensor Rainfall Sensor

Overview

Agriculture is a major driver of food security and economic growth across Africa, but most farms still rely on manual, imprecise irrigation. Without real-time visibility into soil and weather conditions, farmers either over-water — wasting a scarce resource — or under-water during dry spells, hurting crop yield. This is most visible in rural and semi-urban farms with limited access to modern monitoring technology.

This project addresses that with a five-node ESP32 sensor network measuring soil moisture, temperature, humidity, rainfall, and tank water level, feeding into a Flask + MySQL backend. A role-based web dashboard turns that data into irrigation decisions in real time, replacing guesswork with measurement.

System Components

Node 1 — DHT22 Temperature & Humidity

The DHT22 sensor measures temperature and humidity of the farm environment to effectively manage soil, water and support irrigation decisions. The sensor uses a capacitive humidity sensor to detect changes in moisture levels in the air, and an NTC thermistor to measure temperature based on variations in electrical resistance.

Node 2 — Ultrasonic Water Level

The ultrasonic sensor measures distance using sound waves, where a pulse is emitted by the trigger pin, travels through air, reflects off the surface of the object, and is then captured by the echo pin. It works by sending ultrasonic waves from its transmitter, which bounce off the surface of the water and return as an echo to the receiver.

Node 3 — Soil Moisture Sensor

The soil moisture sensor works using two conductive probes that detect changes in resistance based on soil moisture. These variations are converted into voltages and mapped into percentage values, acting as the primary irrigation trigger — when moisture drops below a set threshold, the system activates watering. The soil moisture sensor is a key component that primarily defines the irrigation process.

Node 4 — Rainfall Sensor

The rainfall sensor is responsible for measuring the presence of rainfall, which is essential to managing the irrigation process. This resistive rainfall sensor consists of a conduction board. When water lands on the sensor board, it conducts electricity between these traces, causing changes in resistance which is then converted into a voltage signal read by the ESP32 via an analog pin.

Node 5 — PIR Motion Sensor

The motion sensor is implemented to detect the presence of humans or farmers in the field before automatic irrigation is triggered, for safety. It uses infrared radiation heat emitted by warm bodies, and produces a HIGH/LOW digital output based on whether motion is detected.

Technical Challenges & Fixes

ERROR: Nodes intermittently dropping Wi-Fi
Cause: high traffic when all five nodes transmitted at once.
RESOLVED: staggered each node's transmission interval.
ERROR: NULL values appearing in the database
Cause: sensor readings occasionally lost before reaching Flask.
RESOLVED: only insert a row once a valid reading is actually received.
ERROR: Wiring shorting during testing
Cause: loose jumper wires shifting and touching on small vibrations.
RESOLVED: securely mounted and routed wiring along the enclosure edge.
ERROR: Query performance degrading over time
Cause: five sensor tables accumulating rows indefinitely.
RESOLVED: scheduled SQL cleanup query dropping rows past a set age.
